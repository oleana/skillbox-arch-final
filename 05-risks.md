# 5. Первичный анализ рисков

# Назначение документа

Документ содержит предварительный анализ рисков проекта, выявленных на этапе разработки концепции архитектуры.

На данном этапе оцениваются только наиболее существенные бизнес- и технические риски. После детализации архитектуры анализ будет дополнен оценкой вероятности реализации рисков, стоимости их устранения и остаточного риска.

---

# Методика оценки

Для каждого риска оцениваются:

- вероятность возникновения;
- влияние на проект;
- архитектурные меры по снижению риска.

Используется следующая шкала оценки.

| Значение | Описание |
|----------|----------|
| Низкое | Маловероятно или не оказывает существенного влияния |
| Среднее | Возможно при определённых условиях |
| Высокое | Вероятно и оказывает существенное влияние |

# Наиболее критичные риски

На текущем этапе наиболее значимыми являются следующие риски:

### Массовые соревнования

Во время проведения крупных соревнований ожидается кратковременное увеличение нагрузки в несколько раз. Архитектура должна обеспечивать автоматическое масштабирование без деградации пользовательского опыта.

---

### Защита персональных данных

Платформа хранит персональные данные пользователей, сведения о местоположении, маршрутах тренировок и медицинские показатели, полученные с подключённых устройств. Компрометация таких данных может привести к финансовым и репутационным потерям, поэтому безопасность рассматривается как один из ключевых атрибутов качества.

---

### Большое количество внешних интеграций

Платформа взаимодействует с интернет-магазином компании, существующими спортивными приложениями, мобильными платформами и внешними устройствами. Отказ любой из интеграций не должен влиять на выполнение основных пользовательских сценариев.

---

### Высокая скорость развития продукта

Планируется регулярное добавление новых функций и интеграций. Архитектура должна позволять развивать отдельные компоненты независимо друг от друга без необходимости изменения всей системы.
