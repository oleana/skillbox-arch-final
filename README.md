# Sport Community Platform
Финальная работа курса **«Архитектор программного обеспечения»**.

## О проекте
Проект представляет собой глобальную спортивную платформу, объединяющую пользователей по интересам, видам спорта и уровню подготовки.

Приложение позволяет:
- вести историю тренировок;
- анализировать спортивные показатели;
- сравнивать результаты с собой, другими пользователями и профессиональными спортсменами;
- находить единомышленников;
- организовывать совместные тренировки;
- участвовать в соревнованиях;
- получать персональные рекомендации по тренировкам и спортивному инвентарю;
- покупать продукцию компании через существующие сервисы.

При проектировании особое внимание уделено:
- масштабируемости;
- высокой доступности;
- безопасности пользовательских данных;
- интеграции с существующей ИТ-инфраструктурой компании.

# Этапы проекта

## Этап 1. Утверждение концепта
- Бизнес-цели
- Функциональные требования
- Анализ стейкхолдеров
- Концептуальная архитектура
- Первичный анализ рисков
- План реализации
- Критические бизнес-сценарии
- Атрибуты качества
- Нефункциональные требования
- Анализ архитектурных вариантов

## Этап 2. Защита архитектуры
- Architecture Decision Records (ADR)
- Сценарии использования
- Базовая архитектура с учётом ограничений бизнес-требований, НФТ, выбранной архитектуры, адресация атрибутов качества
- Представления архитектуры: a.	Функциональное. b.	Информационное. c.	Многозадачность (concurrency).d.	Инфраструктурное. e.	Безопасность. 

## Этап 3. Финальная защита
- Анализ рисков созданной архитектуры, компромиссов
- Стоимость владения системой (TCO) в первый, второй и пятый годы с учётом роста данных и базы пользователей.
- Презентация архитектурного решения
