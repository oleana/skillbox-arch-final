# 14. Представления архитектуры

# Назначение документа

Документ содержит представления архитектуры системы с различных точек зрения.

Использование нескольких представлений позволяет описать систему для разных заинтересованных сторон: разработчиков, архитекторов, DevOps-инженеров, специалистов по информационной безопасности и эксплуатации.

---

# 14.1 Функциональное представление

## Назначение

Показывает функциональные компоненты системы и их взаимодействие.

Основные домены:

- Identity;
- User Profile;
- Training;
- Search;
- Social;
- Recommendation;
- Equipment;
- Promotion;
- Gamification;
- Analytics;
- Notification;
- Integration.

Каждый домен реализован отдельным микросервисом и отвечает только за собственную бизнес-логику.

Диаграмма:

- [functional-view.puml](/diagrams/functional-view.puml)

---

# 14.2 Информационное представление

## Назначение

Показывает основные потоки данных и правила хранения информации.

Используются следующие принципы:

- Database per Service;
- Event Driven Architecture;
- Data Lake для аналитики;
- Object Storage для медиа;
- Redis для кэширования.

Основные информационные объекты:

- пользователь;
- профиль;
- тренировка;
- маршрут;
- показатели устройств;
- соревнование;
- достижение;
- рекомендация;
- акция.

Диаграмма:

- [information-view.puml](/diagrams/information-view.puml)

---

# 14.3 Представление многозадачности (Concurrency View)

## Назначение

Показывает параллельную обработку пользовательских запросов и событий.

Используются:

- Kubernetes;
- Kafka;
- асинхронная обработка;
- независимое масштабирование сервисов;
- Horizontal Pod Autoscaler.

Параллельно могут выполняться:

- обработка тренировок;
- обновление рейтингов;
- аналитика;
- генерация рекомендаций;
- отправка уведомлений.

Диаграмма:

- [concurrency-view.puml](/diagrams/concurrency-view.puml)

---

# 14.4 Инфраструктурное представление

## Назначение

Показывает размещение компонентов системы.

Используются:

- CDN;
- Global Load Balancer;
- Multi-region Kubernetes;
- PostgreSQL;
- Kafka Cluster;
- Redis Cluster;
- Object Storage;
- Data Lake.

Платформа развёртывается в нескольких регионах и поддерживает автоматическое переключение между ними.

Диаграмма:

- [infrastructure-view.puml](/diagrams/infrastructure-view.puml)

---

# 14.5 Представление безопасности

## Назначение

Показывает механизмы обеспечения безопасности платформы.

Основные механизмы:

- OAuth2/OpenID Connect;
- JWT;
- RBAC;
- mTLS;
- API Gateway;
- TLS;
- аудит действий;
- централизованное логирование;
- управление секретами.

Безопасность реализуется по принципу Zero Trust.

Диаграмма:

- [security-view.puml](/diagrams/security-view.puml)
